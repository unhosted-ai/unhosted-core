# `models/distill/` — train your own small specialist

Scaffolding for distilling a small open-source base model into a sharper, task-specific version that can run locally through unhosted.

**Status:** scaffold only. Nothing runs without you (a) installing the Python deps and (b) supplying a GPU or accepting CPU-class speeds. Real training is the user's call — this directory ships the recipe, not the run.

## Why distillation, not pretraining

Pretraining a 7B model from scratch costs ~$200K of H100 time. Pretraining a 1B model from scratch is still ~$20K. Distillation — using a strong "teacher" model to generate (prompt, ideal-response) pairs and SFT-ing a small student on them — costs **single-digit dollars** and produces a model that's narrowly excellent at the task you cared about. For a daemon whose value prop is *runs on the hardware you already own*, narrow excellence on a tiny base beats mediocre breadth on a big one.

## What this recipe targets

- **Base:** `TinyLlama/TinyLlama-1.1B-Chat-v1.0`. Llama-architecture, 1.1B params, trained on 3T tokens. Small enough to iterate in ~30 minutes per training run on a consumer GPU; small enough to run quantized on a laptop afterwards.
- **Method:** QLoRA — 4-bit quantized base + low-rank adapter. The base weights stay frozen and quantized; only the LoRA adapter trains. Memory budget: ~8 GB VRAM with default settings.
- **Task (default):** "answer questions about a small document set" — a synthetic Q&A-over-docs corpus generated by a teacher model. Easy to evaluate, matches unhosted's local-memory story, generalizes to many real use cases by swapping the document set.

## Quickstart

```bash
cd models/distill
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 1. Build a dataset. Three ways:
#
#    a) Generate your own from a teacher (Opus 4.8 recommended):
#         python gen_data.py --docs docs/ --out data/train.jsonl --model claude-opus-4-8
#
#    b) Start from an existing teacher dataset on the Hub:
#         python from_hf.py --dataset Roman1111111/claude-opus-4.6-10000x \
#             --out data/train.jsonl --include-reasoning
#
#    c) Hand-supply `data/train.jsonl` with one {"prompt": ..., "response": ...} per line.

# 2. Train. Defaults: TinyLlama-1.1B-Chat, 3 epochs, LoRA r=16.
python train.py --data data/train.jsonl --out out/adapter

# 3. Inference smoke-test (loads adapter, prompts the model, prints output).
python train.py --inference --adapter out/adapter --prompt "your question here"
```

| Apple M2 Pro (MPS) | ~30–60 min | ~1–2s |

| CPU only | hours; don't | ~10s |

## What this is NOT

- **A turnkey product.** A clean recipe with sensible defaults. Real distillation projects need their own teacher, their own data, their own eval set.
- **A way to bypass training compute.** QLoRA cuts memory, not training time meaningfully. A 5K-pair run is fast because 5K is small.
- **An endorsement of TinyLlama for production.** It's a fine dev/test base for proving the pipeline. For a real specialist you'd start from Qwen2.5-7B or Llama-3.1-8B.
