            quick start

            endpoints

            • POST /v1/chat/completions — OpenAI-compatible chat. Set stream: true for SSE token streaming.
            • GET /v1/models — models the upstream is serving.
            • GET /v1/chats — your conversation history (read), DELETE to clear.
            • GET /v1/status — daemon + upstream health.

            non-loopback calls (i.e. over the tunnel) require Authorization: Bearer <token>. loopback is allowed without auth so localhost tooling just works.

            private memory

            summaries of past conversations, stored at ~/.config/unhosted/memories.json. each new chat retrieves the most relevant ones and feeds them to the model as system context — never leaves this machine. delete any entry you don't want the model to see.

            • no memories yet — start chatting with memory on, then save the chat as a memory.

            vram-pool capability

            VRAM-pooling shards one model's layers across multiple LAN peers via llama.cpp's RPC mode, so a 70B-class model becomes usable across hardware no single machine could fit. This panel reports what this machine currently has installed. Orchestration commands ship in v0.1.0 — until then, unhosted vram-pool detect is the CLI equivalent.
